### 位运算优先级的坑

位运算的优先级并不高，低于==。

以下优先级从高到低：
```
乘除取余加减

移位

< <= > >=

== !=

位运算

&& ||

```
由于位运算优先级相对较低，因此在C++里位运算会有这样的坑：
```C++
if ((j & k) == 0)
```
如果写成了
```C++
if (j & k == 0)
```
这里==会优先于&先算，然后再与j相&，不是想要的逻辑。

在`AcWing292. 炮兵阵地`和`LeetCode1276. 不浪费原料的汉堡制作方案`中都有遇到这样的坑，很隐蔽。

Java里运算符优先级也是这样，k == 0会先算，但是会因为int & boolean报错。

总之，有用到位运算和移位运算符时，最好打括号把那部分括起来。
